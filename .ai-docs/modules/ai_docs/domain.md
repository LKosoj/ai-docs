# ai_docs/domain

```markdown
Модуль предоставляет функции для классификации файлов по типам (например, код, документация, конфигурации) и определения доменов инфраструктуры на основе имени файла, расширения и содержимого.  
Поддерживает анализ по расширениям, именам файлов и путям, а также распознаёт специфичные инфраструктурные технологии: Docker, Kubernetes, Helm, Terraform, Ansible и CI/CD.  
Используется для автоматической категоризации файлов в проектах, особенно в контексте анализа репозиториев.

CODE_EXTENSIONS
Набор расширений файлов исходного кода.
---
DOC_EXTENSIONS
Набор расширений файлов документации.
---
CONFIG_EXTENSIONS
Набор расширений конфигурационных файлов.
---
DATA_EXTENSIONS
Набор расширений файлов данных.
---
K8S_FILENAMES
Набор имён файлов, связанных с Kubernetes.
---
CI_FILENAMES
Набор имён файлов конфигураций CI/CD.
---
HELM_FILENAMES
Набор имён файлов, связанных с Helm.
---
DOCKER_FILENAMES
Набор имён файлов, связанных с Docker.
---
TERRAFORM_EXTENSIONS
Набор расширений файлов Terraform.

---
classify_type(path: Path) -> str
Определяет тип файла на основе имени, расширения и пути.
Аргументы
path — путь к файлу для классификации
Возвращает
Строка, описывающая тип файла: "code", "docs", "config", "data", "infra", "ci" или "other"

---
detect_domains(path: Path, content_snippet: str) -> Set[str]
Определяет домены инфраструктуры, к которым относится файл, на основе имени, пути и фрагмента содержимого.
Аргументы
path — путь к файлу
content_snippet — фрагмент содержимого файла (может быть пустым)
Возвращает
Множество строк, каждая из которых представляет обнаруженный домен (например, "docker", "kubernetes")

---
is_infra(domains: Set[str]) -> bool
Проверяет, относится ли файл к инфраструктурным доменам.
Аргументы
domains — множество доменов, возвращённое функцией detect_domains
Возвращает
True, если хотя бы один из доменов относится к инфраструктуре, иначе False
```
