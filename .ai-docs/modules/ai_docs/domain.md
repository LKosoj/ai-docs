# ai_docs/domain

```markdown
Модуль предоставляет функции для классификации файлов по типу и определения их принадлежности к тем или иным областям (доменам) инфраструктуры, кода, конфигураций и других категорий.  
Он использует расширения файлов, имена файлов и пути для определения типа содержимого.  
Также поддерживается обнаружение доменов на основе контекста пути и фрагмента содержимого.

Ключевые структуры данных
CODE_EXTENSIONS — набор расширений файлов исходного кода
DOC_EXTENSIONS — набор расширений файлов документации
CONFIG_EXTENSIONS — набор расширений конфигурационных файлов
DATA_EXTENSIONS — набор расширений файлов данных
TERRAFORM_EXTENSIONS — набор расширений файлов Terraform
K8S_FILENAMES — набор имён файлов, связанных с Kubernetes
CI_FILENAMES — набор имён файлов CI-конфигураций
HELM_FILENAMES — набор имён файлов Helm-чартов
DOCKER_FILENAMES — набор имён файлов, связанных с Docker

---
def classify_type(path: Path) -> str
Классифицирует файл по его пути на одну из категорий: code, docs, config, data, infra, ci, other.
Аргументы
path — путь к файлу для классификации
Возвращает
Строка с типом файла: "code", "docs", "config", "data", "infra", "ci" или "other"

---
def detect_domains(path: Path, content_snippet: str) -> Set[str]
Определяет домены (инфраструктурные технологии), к которым может относиться файл, на основе имени, пути и фрагмента содержимого.
Аргументы
path — путь к файлу
content_snippet — фрагмент содержимого файла для анализа
Возвращает
Набор строк с названиями доменов, например {"docker", "kubernetes"}

---
def is_infra(domains: Set[str]) -> bool
Проверяет, относится ли файл к инфраструктурным доменам.
Аргументы
domains — набор доменов, полученный из detect_domains
Возвращает
True, если хотя бы один из доменов относится к инфраструктуре, иначе False
```
