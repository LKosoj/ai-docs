# ai_docs/domain

```markdown
Модуль предоставляет функции для классификации типов файлов и определения доменов инфраструктуры на основе имени, расширения и содержимого файла.  
Используется для анализа структуры проекта, особенно в контексте DevOps и облачных приложений.  
Поддерживает распознавание кода, конфигураций, документации, данных и специализированных инфраструктурных компонентов.

Ключевые структуры данных  
Path — объект пути к файлу из стандартной библиотеки pathlib  
Set — коллекция для хранения уникальных доменов, возвращаемая функцией detect_domains

classify_type(path: Path) -> str  
Определяет тип файла на основе имени, расширения и пути.  
Аргументы  
path — путь к файлу  
Возвращает  
Строка, описывающая тип файла: "code", "docs", "config", "data", "ci", "infra" или "other"  
Исключения  
Не выбрасывает исключения

---
detect_domains(path: Path, content_snippet: str) -> Set[str]  
Определяет домены, к которым относится файл, на основе имени, пути и фрагмента содержимого.  
Аргументы  
path — путь к файлу  
content_snippet — фрагмент содержимого файла (может быть пустым)  
Возвращает  
Множество строк с названиями доменов (например, "kubernetes", "docker", "ci")  
Исключения  
Не выбрасывает исключения

---
is_infra(domains: Set[str]) -> bool  
Проверяет, относится ли файл к инфраструктурным компонентам по наличию соответствующих доменов.  
Аргументы  
domains — множество доменов, определённых ранее  
Возвращает  
True, если хотя бы один домен относится к инфраструктуре, иначе False  
Исключения  
Не выбрасывает исключения
```
