# ai_docs/generator_cache

```markdown
Модуль предоставляет функции для управления кэшированием, сравнения файлов, построения карты файлов и управления промежуточными результатами (например, аннотациями и сводками).  
Он интегрируется с `CacheManager` для хранения состояния файлов и LLM-кэша, а также обеспечивает синхронизацию данных между запусками.  
Ключевая логика включает вычисление различий, восстановление существующих результатов и очистку устаревших артефактов.

Ключевые структуры данных
file_map — словарь, сопоставляющий путь файла с его метаданными: хешем, размером, типом, доменами и содержимым
index_data — словарь с информацией о предыдущем состоянии индекса, включая файлы и секции

init_cache(cache_dir: Path, use_cache: bool)
Инициализирует менеджер кэша и загружает сохранённые данные
Аргументы
cache_dir — директория для хранения кэша
use_cache — флаг, указывающий, следует ли использовать кэширование LLM
Возвращает
Кортеж: (cache, llm_cache, index_data, prev_files), где llm_cache может быть None
Исключения
Может выбрасывать исключения, связанные с чтением файлов или десериализацией данных

---
build_file_map(files: List[Dict]) -> Dict[str, Dict]
Создаёт карту файлов с хешами и метаданными из списка файлов
Аргументы
files — список словарей с полями path, content, size, type, domains
Возвращает
Словарь, индексированный путями, содержащий хеш, размер, тип, домены и содержимое
Исключения
Не выбрасывает исключения при корректных входных данных

---
diff_files(cache: CacheManager, file_map: Dict[str, Dict])
Вычисляет различия между текущей картой файлов и данными в кэше
Аргументы
cache — экземпляр CacheManager
file_map — текущая карта файлов
Возвращает
Словарь с ключами 'added', 'changed', 'unchanged', 'deleted', содержащий соответствующие пути
Исключения
Может выбрасывать исключения при отсутствии ожидаемых данных в кэше

---
ensure_summary_dirs(cache_dir: Path)
Создаёт директории для хранения промежуточных сводок (файлов, модулей, конфигов)
Аргументы
cache_dir — корневая директория кэша
Возвращает
Кортеж из путей: (summaries_dir, module_summaries_dir, config_summaries_dir)
Исключения
Может выбрасывать исключения при ошибках создания директорий

---
save_cache_snapshot(cache: CacheManager, file_map: Dict[str, Dict], index_data: Dict, llm_cache: Dict[str, str], use_cache: bool) -> None
Сохраняет снимок состояния файлов и кэша LLM (если включено)
Аргументы
cache — экземпляр CacheManager
file_map — текущая карта файлов
index_data — данные индекса, включая секции
llm_cache — кэш ответов LLM
use_cache — флаг использования кэша
Исключения
Может выбрасывать исключения при ошибках записи в файловую систему

---
carry_unchanged_summaries(unchanged: Dict[str, Dict], prev_files: Dict[str, Dict]) -> Tuple[List[Tuple[str, Dict]], List[Tuple[str, Dict]], List[Tuple[str, Dict]]]
Переносит пути к существующим сводкам для неизменённых файлов или определяет, какие нужно перегенерировать
Аргументы
unchanged — словарь неизменённых файлов
prev_files — словарь файлов из предыдущего запуска
Возвращает
Кортеж из трёх списков: файлы без summary, без module_summary, без config_summary
Исключения
Не выбрасывает исключения при корректных входных данных

---
cleanup_orphan_summaries(file_map: Dict[str, Dict], summaries_dir: Path, module_summaries_dir: Path, config_summaries_dir: Path) -> None
Удаляет сводки, на которые больше нет ссылок в текущей карте файлов
Аргументы
file_map — текущая карта файлов
summaries_dir — директория сводок по файлам
module_summaries_dir — директория сводок по модулям
config_summaries_dir — директория сводок по конфигурациям
Исключения
Может выбрасывать исключения при ошибках удаления файлов

---
cleanup_deleted_summaries(deleted: Dict[str, Dict]) -> None
Удаляет сводки для файлов, которые были удалены из проекта
Аргументы
deleted — словарь удалённых файлов с их метаданными
Исключения
Может выбрасывать исключения при ошибках удаления файлов
```
