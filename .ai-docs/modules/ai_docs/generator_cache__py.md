# ai_docs/generator_cache

```markdown
Модуль предоставляет инструменты для управления кэшированием, сравнения файлов, обновления и очистки промежуточных данных (например, резюме файлов) в рамках процесса генерации документации.  
Он взаимодействует с `CacheManager` для хранения состояний файлов и LLM-кэша, строит карту файлов с метаданными, выявляет изменения и обеспечивает актуальность кэшированных артефактов.  
Также реализована логика восстановления неизменённых резюме и удаления устаревших или неиспользуемых файлов подсказок.

Ключевые структуры данных
file_map — словарь, сопоставляющий путь файла с его хэшем, размером, типом, доменами и содержимым
prev_files — словарь предыдущих метаданных файлов из кэша, используемый для сравнения изменений

init_cache(cache_dir: Path, use_cache: bool)
Инициализирует менеджер кэша и загружает сохранённые данные (индекс и LLM-кэш) при необходимости
Аргументы
cache_dir — путь к директории кэша
use_cache — флаг, указывающий, следует ли использовать кэширование
Возвращает
Кортеж: (cache, llm_cache, index_data, prev_files), где cache — экземпляр CacheManager, llm_cache — кэш LLM (или None), index_data — данные индекса, prev_files — предыдущие метаданные файлов
Исключения
Выбрасывает исключение при ошибках чтения кэша или повреждении данных

---
build_file_map(files: List[Dict]) -> Dict[str, Dict]
Создаёт карту файлов с хэшами и метаданными на основе списка файлов
Аргументы
files — список словарей с полями "path", "content", "size", "type", "domains"
Возвращает
Словарь, где ключ — путь файла, значение — метаданные (хэш, размер, тип, домены, содержимое)

---
diff_files(cache: CacheManager, file_map: Dict[str, Dict])
Выполняет сравнение текущей карты файлов с сохранённым состоянием в кэше
Аргументы
cache — экземпляр CacheManager
file_map — текущая карта файлов с метаданными
Возвращает
Словарь с результатами сравнения: добавленные, изменённые, неизменённые и удалённые файлы

---
ensure_summary_dirs(cache_dir: Path)
Создаёт директории для хранения промежуточных резюме файлов, модулей и конфигураций
Аргументы
cache_dir — путь к основной директории кэша
Возвращает
Кортеж из путей: (summaries_dir, module_summaries_dir, config_summaries_dir)

---
save_cache_snapshot(cache: CacheManager, file_map: Dict[str, Dict], index_data: Dict, llm_cache: Dict[str, str], use_cache: bool) -> None
Сохраняет снимок текущего состояния файлов и кэша LLM в постоянное хранилище
Аргументы
cache — экземпляр CacheManager
file_map — текущая карта файлов (без содержимого)
index_data — данные индекса, включая секции
llm_cache — кэш ответов LLM
use_cache — флаг использования кэша; если False, LLM-кэш не сохраняется

---
carry_unchanged_summaries(unchanged: Dict[str, Dict], prev_files: Dict[str, Dict]) -> Tuple[List[Tuple[str, Dict]], List[Tuple[str, Dict]], List[Tuple[str, Dict]]]
Переносит пути к существующим резюме для неизменённых файлов или определяет, какие резюме нужно перегенерировать
Аргументы
unchanged — словарь неизменённых файлов
prev_files — предыдущие метаданные файлов из кэша
Возвращает
Кортеж из трёх списков: файлы без резюме, без резюме модулей и без резюме конфигураций

---
cleanup_orphan_summaries(file_map: Dict[str, Dict], summaries_dir: Path, module_summaries_dir: Path, config_summaries_dir: Path) -> None
Удаляет резюме, которые больше не ссылаются ни на один файл («затерянные» файлы)
Аргументы
file_map — текущая карта файлов с актуальными метаданными
summaries_dir — директория с обычными резюме
module_summaries_dir — директория с резюме модулей
config_summaries_dir — директория с резюме конфигураций

---
cleanup_deleted_summaries(deleted: Dict[str, Dict]) -> None
Удаляет файлы резюме для удалённых исходных файлов
Аргументы
deleted — словарь удалённых файлов с их метаданными (включая пути к резюме)
```
