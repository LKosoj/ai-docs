# tests/test_cache

```markdown
Модуль предоставляет тесты для класса `CacheManager`, проверяя корректность определения изменений в файлах (добавление, изменение, удаление, неизменённые) при сравнении текущего состояния с сохранённым в кэше. Использует временные каталоги для изоляции тестов и безопасной работы с файловой системой. Тесты охватывают сценарии первого запуска, обновления файлов и удаления файлов.

test_diff_files(self)
Проверяет корректность определения изменений в файлах: добавленные, изменённые, удалённые и неизменённые.
Аргументы
self — экземпляр тестового класса
---
CacheManager
Управляет кэшированием состояния файлов и секций документации, обеспечивая сравнение и сохранение индекса.
Поля
cache_dir — каталог, в котором хранится кэш индекса
Методы
__init__(self, cache_dir: Path) — инициализирует менеджер кэша с указанным каталогом
save_index(self, index: dict) — сохраняет индекс в файл кэша
diff_files(self, current_files: dict) -> tuple[dict, dict, dict, dict] — сравнивает текущие файлы с сохранёнными и возвращает наборы добавленных, изменённых, удалённых и неизменённых файлов
```
