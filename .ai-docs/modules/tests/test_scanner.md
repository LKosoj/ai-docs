# tests/test_scanner

```markdown
Модуль `scanner` предоставляет функциональность для рекурсивного сканирования файлов в указанной директории с поддержкой фильтрации по правилам `.gitignore` и исключения системных или временных файлов. Он анализирует структуру проекта, определяет значимые исходные файлы и возвращает их содержимое вместе с метаданными. Основной входной точкой является функция `scan_source`, которая обрабатывает как локальные директории, так и удалённые репозитории в будущем.

scan_source(path: str) -> dict
Сканирует указанную директорию и возвращает структуру с информацией о найденных файлах, исключая игнорируемые согласно .gitignore и внутренним правилам.
Аргументы
path — Путь к директории, которую необходимо просканировать.
Возвращает
Словарь, содержащий список файлов с полями: "path" (относительный путь), "content" (содержимое файла как строка), а также служебную информацию о процессе сканирования.

---
ScannerTests
Тестовый класс для проверки корректности работы функции scan_source.
Методы
test_scan_local_dir — Проверяет, что scan_source корректно сканирует локальную директорию, включая обработку .gitignore и исключение виртуальных окружений.
```
