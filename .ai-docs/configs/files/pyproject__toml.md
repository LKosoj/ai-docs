# pyproject.toml

Конфигурационный файл `pyproject.toml` определяет метаданные проекта, зависимости, настройки сборки и точки входа для CLI-инструмента. Он используется системами сборки Python для корректной упаковки и установки пакета. Файл соответствует стандарту PEP 518 и поддерживает современные практики управления зависимостями и сборкой.

Секции и ключи

build-system — указывает требования и бэкенд для сборки проекта<br>
build-system.requires — список зависимостей, необходимых для сборки пакета<br>
build-system.build-backend — реализация бэкенда сборки, используемая при упаковке<br>
project — содержит основные метаданные проекта<br>
project.name — имя пакета, публикуемое в репозитории<br>
project.version — текущая версия пакета<br>
project.description — краткое описание проекта<br>
project.readme — путь к файлу с подробным описанием проекта<br>
project.requires-python — минимальная версия Python, необходимая для работы пакета<br>
project.dependencies — список зависимостей времени выполнения<br>
project.scripts — определяет исполняемые команды и соответствующие им функции<br>
tool.setuptools.packages.find — настройки автоматического поиска пакетов<br>
tool.setuptools.packages.find.include — явное указание, какие пакеты включать в сборку

Важные параметры

build-backend — критически важен для определения инструмента, собирающего пакет<br>
project.dependencies — определяет внешние библиотеки, от которых зависит работа приложения<br>
project.scripts — позволяет запускать приложение из командной строки после установки<br>
requires-python — гарантирует совместимость пакета с определёнными версиями интерпретатора
