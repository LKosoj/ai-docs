# pyproject.toml

Конфигурационный файл `pyproject.toml` определяет метаданные проекта, зависимости, параметры сборки и инструменты для пакета Python. Он используется системами сборки, такими как `setuptools`, для корректной упаковки и установки проекта. Файл содержит информацию о версии, внешних зависимостях, точках входа и настройках пакета.

Секции и ключи

build-system — указывает требования и бэкенд для сборки пакета<br>
build-system.requires — список зависимостей, необходимых для сборки проекта<br>
build-system.build-backend — имя бэкенда, используемого для сборки (в данном случае — setuptools)<br>
project — содержит основные метаданные проекта<br>
project.name — имя пакета<br>
project.version — текущая версия пакета<br>
project.description — краткое описание пакета<br>
project.readme — путь к файлу с описанием проекта в формате Markdown<br>
project.requires-python — минимальная версия Python, необходимая для работы пакета<br>
project.dependencies — список зависимостей времени выполнения<br>
project.scripts — определяет исполняемые команды и их точки входа<br>
tool.setuptools.packages.find — настройки автоматического поиска пакетов<br>
tool.setuptools.packages.find.include — список шаблонов имён пакетов для включения в сборку<br>
tool.setuptools.packages.find.exclude — список шаблонов имён пакетов, исключаемых из сборки<br>
tool.setuptools.package-data — указывает дополнительные файлы, которые должны быть включены в пакет<br>
tool.setuptools.package-data.ai_docs — файлы данных, включаемые в пакет ai_docs

Важные параметры

build-backend — критически важен для определения инструмента сборки, должен соответствовать используемой системе<br>
project.dependencies — определяет сторонние библиотеки, от которых зависит работа приложения<br>
project.scripts — позволяет создавать CLI-команды, доступные после установки пакета<br>
tool.setuptools.package-data — обеспечивает включение статических файлов, необходимых в runtime
