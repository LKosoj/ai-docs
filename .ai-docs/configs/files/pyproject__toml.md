# pyproject.toml

Конфигурационный файл `pyproject.toml` определяет метаданные проекта, зависимости, настройки сборки и точки входа для CLI-инструмента. Он используется системами сборки Python для корректной упаковки и установки проекта. Файл соответствует стандарту PEP 518 и поддерживает современные практики управления зависимостями и сборкой.

Секции и ключи

build-system — указывает требования и бэкенд для сборки проекта<br>
build-system.requires — список зависимостей, необходимых для сборки пакета<br>
build-system.build-backend — имя бэкенда, используемого для сборки (в данном случае setuptools)<br>
project — содержит основные метаданные проекта<br>
project.name — имя пакета<br>
project.version — версия пакета<br>
project.description — краткое описание пакета<br>
project.readme — путь к файлу с описанием проекта в формате Markdown<br>
project.requires-python — минимальная версия Python, необходимая для работы пакета<br>
project.dependencies — список зависимостей времени выполнения<br>
project.scripts — определяет исполняемые команды и их точки входа<br>
project.scripts.ai-docs — команда, доступная в терминале, и соответствующая функция запуска<br>
tool.setuptools.packages.find — настройки автоматического поиска пакетов<br>
tool.setuptools.packages.find.include — явное указание, какие пакеты включать в сборку

Важные параметры

project.name — должно быть уникальным и соответствовать имени, под которым пакет будет опубликован<br>
project.scripts — позволяет создавать удобные CLI-команды для пользователей после установки пакета<br>
build-system.build-backend — критически важен для определения инструмента сборки, используемого при установке пакета
