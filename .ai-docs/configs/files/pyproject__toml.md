# pyproject.toml

Конфигурационный файл `pyproject.toml` определяет метаданные проекта, зависимости, параметры сборки и инструменты для пакета Python. Он используется системами сборки, такими как `setuptools`, для корректной упаковки и установки проекта. Файл содержит как стандартные секции, определённые спецификацией, так и расширения от конкретных инструментов.

Секции и ключи

build-system — указывает требования и бэкенд для сборки пакета<br>
build-system.requires — список зависимостей, необходимых для сборки проекта<br>
build-system.build-backend — имя бэкенда, отвечающего за сборку пакета<br>
project — содержит основные метаданные проекта<br>
project.name — имя пакета<br>
project.version — текущая версия пакета<br>
project.description — краткое описание пакета<br>
project.readme — путь к файлу с описанием проекта в формате Markdown<br>
project.requires-python — минимальная версия Python, необходимая для работы пакета<br>
project.dependencies — список зависимостей времени выполнения<br>
project.scripts — определяет исполняемые скрипты, доступные после установки пакета<br>
tool.setuptools.packages.find — настройки автоматического поиска пакетов<br>
tool.setuptools.packages.find.include — список шаблонов имён пакетов для включения<br>
tool.setuptools.packages.find.exclude — список шаблонов имён пакетов для исключения<br>
tool.setuptools.package-data — указывает дополнительные файлы, которые должны быть включены в пакет<br>
tool.setuptools.package-data.ai_docs — файлы, включаемые в пакет ai_docs

Важные параметры

name — уникальное имя пакета в репозитории Python (например, PyPI)<br>
version — версия пакета, используется для управления обновлениями<br>
dependencies — ключевой список внешних библиотек, от которых зависит проект<br>
project.scripts — позволяет создавать команды командной строки, привязанные к функциям в коде<br>
requires-python — гарантирует, что пакет будет установлен только на совместимых версиях Python
