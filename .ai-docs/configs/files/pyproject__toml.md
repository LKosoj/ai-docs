# pyproject.toml

Конфигурационный файл `pyproject.toml` определяет метаданные проекта, зависимости и параметры сборки для Python-пакета. Он используется инструментами сборки, такими как `setuptools` и `pip`, для корректной установки и запуска проекта. Файл содержит информацию о версии, зависимостях, скриптах командной строки и настройках упаковки.

Секции и ключи
build-system — указывает зависимости и бэкенд, необходимые для сборки проекта
requires — список пакетов, требуемых для сборки
build-backend — имя бэкенда, используемого для сборки пакета
project — содержит основные метаданные проекта
name — имя пакета
version — текущая версия пакета
description — краткое описание пакета
readme — путь к файлу с описанием проекта в формате Markdown
requires-python — минимальная версия Python, необходимая для работы пакета
dependencies — список зависимостей времени выполнения
project.scripts — определяет исполняемые скрипты командной строки
ai-docs — точка входа для CLI-команды, запускающая функцию main из модуля ai_docs.cli
tool.setuptools.packages.find — настройки автоматического поиска пакетов
include — список шаблонов имён пакетов, которые следует включить в сборку

Важные параметры
name — уникальное имя пакета, используемое при установке через pip
version — версия пакета, соблюдающая семантическое версионирование
dependencies — ключевой список внешних библиотек, от которых зависит функциональность проекта
requires-python — определяет совместимость с версиями интерпретатора Python
build-backend — критически важен для корректной сборки пакета современными инструментами
