```markdown
## Краткое резюме изменений

- **Автоматическое обнаружение изменений**: модуль анализирует хеши файлов и определяет, какие файлы были добавлены, изменены или удалены. При повторном запуске без изменений — пропускает обработку (при включённом кэше).
- **Умное кэширование**: при `use_cache=True` сохраняются:
  - хеши исходных файлов,
  - LLM-суммаризации,
  - сгенерированные разделы документации.
  Повторные запуски с неизменённой кодовой базой завершаются мгновенно.
- **Очистка устаревших файлов**: при удалении или переименовании исходных файлов соответствующие `.md`-файлы в `docs/summaries/` и `docs/modules/` автоматически удаляются.
- **Параллельная обработка**:
  - Файлы обрабатываются параллельно через `ThreadPoolExecutor` (управляется параметром `threads`).
  - Генерация разделов выполняется асинхронно (`section_workers`), ускоряя сборку многоразделной документации.
- **Актуализация структуры MkDocs**: `mkdocs.yml` перегенерируется на каждом запуске с актуальной навигацией, включая новые модули и доменные разделы.
- **Поддержка доменных разделов**: автоматически создаются разделы на основе подпапок в `configs/`, например, `configs/auth/` → `docs/auth/`.
- **Генерация диаграмм**: в раздел "Архитектура" добавляется Mermaid-диаграмма зависимостей (если не указана вручную).
- **Транслитерация заголовков**: кириллические названия безопасно конвертируются в slug-формат для корректных URL в MkDocs.
- **Управление зависимостями**: автоматически извлекаются из `pyproject.toml`, `requirements.txt`, `package.json` и включаются в `dependencies.md`.
- **Фильтрация тестов**: файлы из путей `test`, `tests`, `__tests__` исключаются из модульной документации через `_is_test_path`.
- **Принудительная перегенерация**: при `force=True` все файлы пересоздаются, игнорируя кэш и хеши.
```
