pyproject.toml

Конфигурационный файл pyproject.toml определяет метаданные проекта, зависимости, настройки сборки и точки входа для CLI-инструмента. Он используется системами сборки Python для корректной упаковки и установки пакета. Файл соответствует стандарту PEP 518 и поддерживает современные практики управления зависимостями и сборкой.

Секции и ключи

build-system — указывает требования и бэкенд для сборки проекта
build-system.requires — список зависимостей, необходимых для сборки пакета
build-system.build-backend — реализация бэкенда сборки, используемая при упаковке
project — содержит основные метаданные проекта
project.name — имя пакета, публикуемое в репозитории
project.version — текущая версия пакета
project.description — краткое описание проекта
project.readme — путь к файлу с подробным описанием проекта
project.requires-python — минимальная версия Python, необходимая для работы пакета
project.dependencies — список зависимостей времени выполнения
project.scripts — определяет исполняемые команды и соответствующие им функции
tool.setuptools.packages.find — настройки автоматического поиска пакетов
tool.setuptools.packages.find.include — явное указание, какие пакеты включать в сборку

Важные параметры

build-backend — критически важен для определения инструмента, собирающего пакет
project.dependencies — определяет внешние библиотеки, от которых зависит работа приложения
project.scripts — позволяет запускать приложение из командной строки после установки
requires-python — гарантирует совместимость пакета с определёнными версиями интерпретатора
