## Краткое резюме

При каждом запуске `ai_docs` анализирует изменения в исходных файлах и фиксирует результаты в `docs/changes.md`. Отчёт автоматически генерируется функцией `format_changes_md` и включает:

- **Добавленные файлы** — новые файлы, обнаруженные в проекте.
- **Изменённые файлы** — файлы с изменённым содержимым (по хешу).
- **Удалённые файлы** — отсутствующие в текущей версии.
- **Перегенерированные разделы** — секции документации, пересозданные из-за изменений в контексте.
- **Краткое резюме** — итоговое описание масштаба и характера изменений.

Отчёт обновляется только при реальных изменениях в кодовой базе. Используется инкрементальный подход: пересчитываются только затронутые файлы, остальные сохраняют кэшированные описания. Это обеспечивает быструю регенерацию документации в CI/CD и локальной разработке.
