# Глоссарий

```markdown
# Глоссарий

- **`ai_docs`** — CLI-утилита для автоматической генерации технической документации по исходному коду и конфигурационным файлам. Поддерживает анализ локальных и удалённых Git-репозиториев, распознавание технологических доменов и инкрементальную генерацию с кэшированием.

- **`LLMClient`** — класс для взаимодействия с языковыми моделями (LLM) через API. Поддерживает кэширование, настройку параметров генерации (модель, температура, токены) и загрузку конфигурации из переменных окружения.

- **`scan_source`** — функция сканирования директории или Git-репозитория. Собирает список файлов с учётом `.gitignore`, фильтров включения/исключения и ограничений по размеру. Возвращает структуру `ScanResult` с метаданными.

- **`ScanResult`** — контейнер результата сканирования: включает список файлов, корневой путь, источник и имя репозитория.

- **`classify_type`** — функция определения типа файла (например, `code`, `config`, `docs`, `infra`) на основе расширения и имени.

- **`detect_domains`** — функция анализа файла на предмет принадлежности к технологическим доменам (Kubernetes, Helm, Terraform и др.) по пути, имени и фрагменту содержимого.

- **`is_infra`** — утилита проверки, относится ли файл к инфраструктурным компонентам (на основе набора доменов).

- **`summarize_file`** — функция генерации краткого описания содержимого файла с помощью LLM. Поддерживает чанкирование по токенам и кэширование.

- **`chunk_text`** — разбиение текста на фрагменты заданного размера в токенах с использованием `tiktoken`.

- **`count_tokens`** — подсчёт количества токенов в тексте для указанной модели.

- **`build_mkdocs_yaml`** — формирование конфигурации `mkdocs.yml` на основе структуры документации. Поддерживает локальный режим (`local_site`).

- **`write_docs_files`** — запись Markdown-файлов документации в указанную директорию с созданием промежуточных папок.

- **`CacheManager`** — класс управления кэшем: хранит индекс файлов (`index.json`) и ответы LLM (`llm_cache.json`). Позволяет определить изменения в файлах через `diff_files`.

- **`diff_files`** — метод сравнения текущих файлов с предыдущим состоянием кэша. Возвращает наборы: `added`, `modified`, `deleted`, `unchanged`.

- **`format_changes_md`** — формирование отчёта об изменениях в Markdown-формате, включая добавленные, изменённые, удалённые файлы и перегенерированные разделы.

- **`safe_slug`** — преобразование строки в безопасный идентификатор (слаг) для использования в путях файлов.

- **`is_binary_file`** — определение, является ли файл бинарным (по наличию нулевых байтов в первых 2048 байтах).

- **`is_url`** — проверка, является ли строка URL (начинается с `http://`, `https://`, `git@`).

- **`.ai_docs_cache`** — директория кэша, содержащая `index.json` (метаданные файлов) и `llm_cache.json` (ответы LLM). Используется для инкрементальной обработки.

- **`ai_docs_site`** — выходная директория со статическим сайтом документации, сгенерированным через `mkdocs build`.

- **`local_site`** — режим генерации документации для локального просмотра. Отключает `use_directory_urls` и `site_url` в `mkdocs.yml`.

- **`threads` / `AI_DOCS_THREADS`** — количество параллельных потоков для обработки файлов и запросов к LLM.
```
